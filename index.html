Django

High-level Python Web Framework

Django is a high-level Python web framework that helps you build web applications quickly and securely.

Key Strengths

Batteries-included

ORM, admin panel, authentication, routing, forms, templating, and more are built-in.

MTV Architecture

Model - Template - View separates data (models), UI (templates), and request logic (views).

Good Defaults & Security

CSRF, XSS protection, password hashing and other security features are included.

Scalable & Extensible

Used for small sites and large applications.

Main Components

  • Project: top-level configuration (manage.py, settings.py, urls.py).
  • App: reusable submodule (e.g., blog, accounts).
  • Models: database tables as Python classes (ORM).
  • Views: functions/classes that handle requests and return responses.
  • Templates: HTML files with Django template language for rendering.
  • Admin: auto UI to manage models.
  • Migrations: schema changes tracked & applied with makemigrations/migrate.

Install Django on Windows

These steps assume a Windows PC. I'll include PowerShell and Command Prompt variants.

  1. Download Python 3.11 or 3.12 from python.org and run the installer.
  2. Important: check "Add Python to PATH" while installing.
  3. Verify in a terminal:
python --version
# or if that fails
py -3 --version

Open Command Prompt or PowerShell, then:

# make a folder and enter it
mkdir C:\projects\mysite
cd C:\projects\mysite

# create virtual env
python -m venv venv
# or
py -3 -m venv venv

PowerShell

.\venv\Scripts\Activate.ps1

# If PowerShell blocks scripts, run once as admin:
# Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

Command Prompt

.\venv\Scripts\activate

After activation your prompt will show (venv).

Upgrade pip and install Django

python -m pip install --upgrade pip

# check version

Create a Django project & app

# create a new project "mysite"
django-admin startproject mysite .

# create an app called "blog"
python manage.py startapp blog

(Notice the . after startproject places files in current folder.)

Run initial migrations and start server

python manage.py migrate
python manage.py runserver

Code Examples

mysite/urls.py

from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', include('blog.urls')),  # route root to blog app
]

blog/urls.py (create this file)

from django.urls import path
from . import views

urlpatterns = [
    path('', views.index, name='index'),
]

blog/views.py

from django.shortcuts import render

def index(request):
    return render(request, 'blog/index.html', {'message': 'Hello Django!'})

Template: blog/templates/blog/index.html

<!doctype html>

    <h1>{{ message }}</h1>

Remember to add 'blog' to INSTALLED_APPS in mysite/settings.py.
